# 배열
일반적인 프로그래밍 언어에서 배열을 선언할 때 배열의 크기를 알려준다. 그러면 운영체제는 그 크기만큼의 연속된 빈 공간을 메모리에서 찾아 배열의 메모리 공간으로 할당한다. 배열에 값을 할당하면 그 부분은 할당한 값으로 채워지고, 그렇지 않은 부분은 쓰레기 값이 저장되어 있다. 

운영체제는 배열의 시작 주소만을 기억한다. 그리고 배열의 세 번째 원소에 접근할 땐 시작 주소로부터 두 번째 떨어진 주소에서 데이터를 가져온다. 이처럼 배열의 인덱스 참조는 길이에 상관없이 한 번에 가져오기 때문에 O(1)의 성능을 가진다. 그래서 배열은 읽기, 쓰기, 즉 참조에서 좋은 성능을 보인다.

하지만 데이터의 삽입, 삭제의 성능은 좋지 않다. 

## 삽입, 삭제 성능이 안좋은 이유
만약 크기가 10인 크기의 배열에 하나의 원소를 더 삽입하려 했다고 생각해보자. 그러면 운영체제는 크기가 11인 크기의 연속된 메모리 공간을 찾아서 할당하고, 기존의 배열의 원소값을 복사해서 새로 할당한 공간에 붙여넣어야 한다. 처음부터 큰 크기의 메모리 공간을 할당하면 문제가 없지 않을까 싶지만, 메모리 공간 낭비가 될 가능성이 있다.
