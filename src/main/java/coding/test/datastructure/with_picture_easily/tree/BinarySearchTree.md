# 이진 탐색 알고리즘
- 단점
  - 배열이 정렬되어 있어야함(되어 있을 때만 사용 가능)
  - 배열은 데이터 삽입 & 제거 -> 비효율적

# 해시 테이블
- 장점 
  - 데이터 삽입, 제거, 검색이 빠름
- 단점
  - 해시 함수 성능이 안좋다면 충돌로 인해 성능이 급격하게 안좋아짐
  - 데이터가 정렬되어 있지 않고, 테이블을 만들어 놓아야 하기에 메모리가 많이 필요

# 이진 탐색 트리
- 장점
  - 해시 테이블처럼 잦은 데이터 삽입 & 삭제 성능 강력
  - 검색 빠름
  - 데이터를 정렬해 저장
  - 데이터를 저장할 공간을 제외한 다른 공간이 불필요
    - 메모리 사용량 적음

- 이진 트리로 구성됨, 몇 가지 규칙이 추가되면 이진 탐색 트리가 됨
  - 규칙 
    - 중복된 노드가 없어야 함
    - 특정 노드의 왼쪽 자식 노드는 그 노드의 값보다 항상 작은 값들로만 이루어짐
    - 특정 노드의 오른쪽 자식 노드는 그 노드의 값보다 항상 큰 값들로만 이루어짐
    - 자식 트리(노드)에도 위의 규칙이 모두 적용되어야 함


- 이진 탐색 트리의 기능
  - 데이터 삽입
  - 데이터 제거
  - 데이터 탐색

## 평가
- 이진 트리는 컴파일러가 사용하는 구문 트리를 만들거나 jpeg, mpeg 파일을 만드는 압축 알고리즘인 허프만 트리를 만들 때 유용하게 쓰임
- 트리만 잘 만들어졌다면 삽입, 제거, 탐색의 기능이 아주 뛰어남
  - 삽입, 제거, 탐색 성능은 O(N)이 될수도 O(logN)이 될수도 있음
- O(logN)의 성능을 보이기 위해서는 '완전이진트리'나 '포화이진트리'처럼 트리가 **균형**을 이루고 있어야 함


- 컴퓨터 과학자들은 이진 탐색 트리의 균형을 어떻게 유지할지 고민
  - 균형을 무너뜨릴 수 있는 상황을 파악한 후 그 상황들에서 균형을 유지하기로 함
  - 데이터 삽입 or 제거 시 균형이 깨질 수 있음
  - 삽입 or 제거 시 균형을 유지하는 방법을 구상해 AVL 트리, 레드-블랙 트리 등을 만듬
    - 이를 '자가 균형 이진 탐색 트리' 라고 부름
