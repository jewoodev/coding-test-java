# P-NP 문제란?
알고리즘의 성능을 평가하는데 쓰이는 'Big-O 표기법'과 달리 P-NP 문제는 '어떤 문제가 주어졌을 때 그 문제가 쉬운 문제인지 어려운 문제인지, 더 나아가 해결이 가능하다고 알려졌는지 아닌지를 판단하는 척도' 가 된다. 

- 결정 문제 
  - 어떤 문제가 주어졌을 때 Yes 또는 No로 대답할 수 있는 문제를 말한다. 
- 최적화 문제 
  - 어떤 상황이 주어지고, 거기서 최적의 해를 찾으라는 문제를 말한다.
  - 대부분의 최적화 문제는 결정 문제로 바꿀 수 있다. 

# P 문제
결정 문제가 주어졌을 때 **결정론적 튜링 머신**으로 **다항 시간 내에** 답을 구할 수 있는 문제를 말한다.

## Deterministic Turing Machine
현재 상태에서 다음 상태로 이동할 때 다음 상태가 유일하게 결정되는 머신이다. 다음 상태에 유일한 상태로 결정되면서 문제를 해결하기 때문에 문제를 일방향으로 해결한다.

- if-else 문은 두 가지 분기로 나뉘니까 결정론적 튜링 머신이 해결하는 방식이 아닌걸까?
  - 아니다. 이건 결튜머가 해결하는 방식이다. 
  - 조건문 수행에 따라 다음 상태가 될 수 있는게 두 가지라 헷갈리지만 현재 상태에서 다음 상태로 넘어갈 때,
    - 즉 if문이나 else문이 수행될 때 다음 상태가 유일하게 결정된다.

오늘날의 컴퓨터가 결튜머의 예이다.

## Polynomial time
- 다항 시간이란 문제를 해결하는데 걸리는 시간이 다항식으로 표현할 수 있는 것을 말함
  - 다항식은 입력 n에 대한 여러 항으로 구성되는 식을 말한다.
- 다항 시간과 상수 시간(O(1)), 로그 시간(O(logN))은 전부 다항 시간 내에 해결할 수 있는 문제임 
- 지수 시간은 다항 시간보다 더 크기 때문에 다항 시간 내에 해결할 수 없음!

# NP 문제
결정 문제가 주어졌을 때 비 결정론적 튜링 머신을 사용해 다항 시간 내에 답을 구할 수 있는 문제

## Nondeterministic Turing Machine
- 현재 상태에서 다음 상태로 넘어갈 때 다음 상태의 개수가 유일하지 않은 머신
- 현실세계에서 볼 수 없고, 상상으로 있다고만 가정한 것
  - 무한한 컴퓨팅 기능이 있다고 가정한 상상의 기계
- 여러 개의 선택지가 있을 때 하나 씩 선택해보지 않고, 분신술을 써서 동시에 선택하는 방식으로 해결
  - 결튜머는 지수 시간이 걸림
    - 지수 시간은 입력이 조금만 커져도 매우 커지기 때문에 현실적으로 해결하기 어려움
    - 다만 정답 후보가 정해지면 정답 후보가 참인지 거짓인지 확인하는 건 결튜머로 다항 시간 내에 검증까진 가능
- 모든 P문제는 NP문제이기도 함
  - 모든 P문제는 비튜머로 다항 시간 내에 해결할 수 있기 때문

# NP-hard 문제
- 어떤 NP 문제들이 있을 때 이 모든 NP 문제들을 다항 시간 내에 '어떤 문제 A'로 환원시킬 수 있다면 그 문제 A를 NP-hard 문제라고 함
- 해결 가능한지, 불가능한지 알려지지 않은 문제

# NP-complete 문제
- NP-hard에도 포함되면서 NP 문제에도 포함되는 문제
- 어떻게든 문제를 해결할 수 있다고 판결이 나온 문제
- e.g) 외판원 문제(TSP)

# P vs NP 문제
수학계에는 밀레니엄 문제라는 게 있다. 이 문제는 7 문제가 있는데 그 중 하나만 증명이 되고 나머지 6 문제는 아직까지 증명되지 않았다. 거기에 'P vs NP 문제'가 있다.

'P vs NP 문제'는 "P 집합과 NP 집합이 같은 집합인가?" 라는 문제이다. 이 문제가 참이라고 밝힌다면 모든 NP 문제는 결튜머로 해결할 수 있다는게 된다.
