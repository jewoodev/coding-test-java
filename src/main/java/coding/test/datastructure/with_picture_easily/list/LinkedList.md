# 연결리스트
배열의 삽입 & 삭제 성능 문제를 해결하기 위해 저장하려는 데이터를 분산시켜 메모리 공간에 할당하고, 데이터를 연결시킨 자료구조이다. 이는 '노드' 라는 걸 만들어 수행하는데, 노드는 데이터를 담는 변수 하나와 다음 노드를 가리키는 변수 하나를 가지고 있다. 이러한 노드로 자료구조를 만들기 때문에 첫 노드로부터 필요한 만큼 다음 노드를 탐색해 자료구조 내의 원소들을 조회할 수 있다. 

연결리스트는 초기 크기를 알 필요가 없다는 장점과 중간 삽입, 삭제 시 재할당 -> 복사 & 붙여넣기를 하지 않고 연결만 시키면 되어 성능이 뛰어나다는 장점이 있다. 대신 자료들이 분산되어 저장되었기 때문에 배열과 다르게 순차적인 탐색이 필요하다, 즉 데이터 참조 성능이 O(N) 이다.

|        | 배열   | 연결리스트 |
|--------|------|-------|
| 크기     | 고정   | 동적    |
| 주소     | 연속   | 불연속   |
| 데이터 참조 | O(1) | O(N)  |
| 삽입과 삭제 | O(N) | O(N)  |

연결리스트의 삽입과 삭제는 처음과 끝에 행할 시 O(1)이지만 중간에 삽입하는 것은 O(N)이다. 삽입하고자 하는 인덱스까지 참조해야 하기 때문이다.
